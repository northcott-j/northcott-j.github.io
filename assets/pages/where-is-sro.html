Where is the SRO?

Background

The Hollis-Brookline SRO is a busy officer traveling between multiple schools and organizing various classes/training for civilians and other officers. Each location has different contact methods and levels of availability. How do you quickly find and contact the SRO if you need them?

Find Functionality

The "public" facing functionality that was needed was a mobile friendly way for specific individuals to be able to see the current status of the SRO.

On the front of the card, users can see the physical location of the SRO. On the back, there is a list of contact information, the time that the status was updated, availability, and an option field for comments to be provided.

When a new user accesses the application with a Google account, they must be approved by an administrator before gaining access to the SRO's status.

Admin Functionality

On the admin side, the functionality needed was updating statuses, adding new users, seeing a log of past statuses, and managing reusable information like Location details.

Technical Architecture

The backend API of the application is written in Python using Flask and Orator ORM. This is hosted on Heroku and uses TravisCI to build deployments. The database is Postgres hosted on Google Cloud.

The frontend for Find and Admin are written in React, deployed via Travis, and hosted on S3. Each is its own repository, but they share the same API backend.

Additional services used are Drift for support, auth0 for authentication, Slack for notifications, Bugsnag for error logging, and Papertrail via Heroku for logging.
