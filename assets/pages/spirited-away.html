Spirited Away

Background

For the Chi Sigma formals (Spirited Away * 2 & Larz Anderson), I needed a ticket purchasing and guest management tool that didn't cost as much as Eventbrite.

So I decided to build my own.

Registration Functionality

The guest sign-up form supports fields to capture information such as discount codes, if a promoter was used, who referred someone to the event, different ticket types, as well as the information you'd expect to be in a ticket purchasing form.

Upon registration, the guest is sent an email with either a link to the Chi Sigma Square Store to complete a credit transaction, or information on how to pay via Venmo. A notification is also sent via Slack to me that someone has registered.

Admin Functionality

As an Admin, I needed to be able to quickly mark people as payment requested/paid, and to send them their tickets. This is done via Slack with API hooks back to the application.

In the UI, I can filter down to individual guests, add notes to their accounts, and send out tickets or other notifications. I can also see stats on the current guest list such as how many people are 21+ or the number of people still waiting to pay.

Technical Architecture

The backend API of the application is written using NodeJS. This is hosted on Heroku and uses TravisCI to build deployments. The database is MongoDB provided by mLab.

The client is hosted on the same Heroku instance as the API and is written using AngularJS.

Additional services used are Slack for notifications and Zoho Mail services for email.
